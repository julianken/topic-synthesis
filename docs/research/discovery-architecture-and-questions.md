# Discovery — recommended architecture & decision tails

## Recommended architecture

Two-phase, code-orchestrated multi-agent pipeline on a durable workflow engine (Trigger.dev/Inngest, step-memoized) calling the Claude API directly with a tiered model strategy (Opus 4.8 for Planner/Graph-builder/Critic; Sonnet 4.6/Haiku 4.5 for researchers and first-draft node synthesis; Message Batches API at -50% for non-latency-sensitive bulk synthesis; prompt-cache the shared graph/brief prefix). ANALYSIS = orchestrator-worker fan-out (STORM-style planning + agentic-RAG researchers with source provenance + graph-builder + a hard grounding-verification gate that also computes per-node coverage_confidence). The prerequisite knowledge graph is the immutable contract between phases (validated by a topo-sort/citation/coverage hard gate before any synthesis), but it is treated as a medium-confidence design bet, not as something Anthropic's research-system evidence validates. SYNTHESIS = map-reduce per graph node: spec-then-code (with an explicit accessibility contract in the spec) + rubric-gated Critic with capped iterations + hub assembler. Thin-coverage nodes degrade to text+citations or stay "soon"; near-zero-coverage topics are surfaced at the review gate (refuse/narrow), never fabricated. A topic-admissibility classifier runs before generation. Postgres is the system-of-record with a SHARE-READY schema from day one (concept_pages keyed by content identity + a curriculum_pages join, workflow_versions carrying their model_snapshots array, append-only telemetry), object storage keyed by content hash, so the per-curriculum-vs-shared-page-library unit is a later config change rather than a rewrite. Every workflow version is a content-hash of DAG+prompts+pinned dated model snapshots+templates and is the A/B arm. Generated artifacts render in a sandboxed, cookieless cross-origin iframe (sandbox="allow-scripts" without allow-same-origin; server-set CSP header; DOMPurify; escape tests). A scheduled model-snapshot-retirement monitor + a cache_read_input_tokens>0 assertion guard against silent model/cache drift invalidating live experiments and shared pages. Chrome is Next.js/React 19/Tailwind v4/shadcn with transitions-dev snippets vendored in; resumable streams (Vercel Workflow DevKit) are a medium-confidence, lock-in-bearing nice-to-have, not a core v1 dependency. Evals split by stage (groundedness/NLI for ANALYSIS; binary-rubric LLM-as-judge with a cross-family judge, an accessibility hard-gate, deterministic render+axe-core checks for SYNTHESIS) on Braintrust; an offline gold set is the primary decision surface given that online learner-outcome A/B is infeasible for most long-tail topics.

## Problem statement

Building a high-quality, interactive, scaffolded curriculum (the hand-built 'ai-concept-viz' artifact: 100+ standalone HTML/Canvas/SVG explainer pages organized into a tiered, status-tracked hub) is artisanal, slow, and expert-dependent. topic-synthesis aims to generate that artifact automatically from a user's topic + settings via a multi-agent ANALYSIS-to-SYNTHESIS workflow. The white space is real and unoccupied (tutors adapt pre-authored corpora; explorable-explanation platforms are hand-built; notebook tools summarize user sources; roadmap builders emit text outlines; single-artifact generation is near-commodity but has no curriculum structure), and the defensible product is the orchestration layer above single artifacts. But the same prior-art finding documents the existential risk: AI-generated educational content confidently fabricates concepts and renders flawed visuals, so the founder's eval + A/B + telemetry requirements are load-bearing. The decision-useful questions the report must answer: a concrete per-curriculum dollar cost (to make the pricing/segment call the report itself calls existential), how to sequence a thin generator vs. an offline gold set vs. online A/B for a solo builder, what happens when ANALYSIS retrieval is thin on niche topics, how delayed learning outcomes can actually be measured on anonymous learners at scale, an honest statistical-power posture given expensive per-observation generation and sparse long-tail traffic, how the per-curriculum-vs-shared-page-library unit decision propagates through cost/schema/A/B, plus the previously-missing angles of generated-artifact accessibility, content-moderation, source-licensing, learner-as-stakeholder trust signals, the SME as an operational constraint, and model-snapshot drift.

## Top design questions

- Unit of generation: per-curriculum vs. shared-page-library keyed by (concept, settings-bucket)? This single choice cascades into the ~$26/curriculum cost (collapsing toward cache-miss count under sharing), the Postgres schema, the A/B randomization unit, AND whether online A/B has enough traffic to ever conclude. Recommendation: keep the schema share-ready from day one even if v1 generates fresh.
- Buyer/segment and price: given ~$26 variable cost per 40-page curriculum, the $4/mo and free anchors are non-starters for per-curriculum consumer pricing. Prosumer/B2B at a price clearing $26 + margin + regeneration allowance, or commit to the shared-library amortization model? This gates the entire business.
- Visual-and-accessibility verification: what concretely certifies a Canvas/SVG page as correct (real math, no fabrication) AND WCAG-conformant (text alternatives, keyboard operability) before it flips from 'soon' to 'built'? Canvas is invisible to assistive tech by default and a WCAG-failing page is a procurement/legal blocker.
- Topic-admissibility policy for a public from-any-topic generator: where is the moderation gate, what are the three outcomes (allow / refuse-with-explanation / allow-with-constraints), how are high-sensitivity domains (medical/legal/financial) framed, and how is age-appropriateness handled as a settings dimension?
- Delayed-outcome measurement mechanics: how do anonymous learners return for a delayed retention test at scale? Spaced-review-as-feature + lightweight returning identity are the only mechanisms likely to work, and even then only for head topics / opted-in learners — what is the within-session surrogate metric for everything else?
- Single north-star learning-outcome metric: delayed retention/transfer is the true verdict but is unreachable for most of the long tail — what composite (offline rubric pass-rate + within-session desirable-difficulty surrogates + delayed retention where available) becomes the actual ship criterion, and how is it protected from the engagement/Goodhart trap?
- Cold-start / thin-coverage routing: what coverage_confidence threshold triggers degrade-to-text vs. keep-as-'soon' vs. refuse-and-narrow at the review gate, and is there a separate retrieval-light workflow for genuinely niche topics (behind an explicit user choice + badge)?
- Spec-vs-page regeneration on Critic failure: regenerate the cheap spec (fixes pedagogy/structure) or the full page (fixes render/run/a11y bugs the spec critique can't see)? This is the largest cost line (~45%) so the policy directly drives unit economics.
- SME operational plan: who is the 'benevolent dictator', what are their committed monthly hours for recalibration + the annotation queue, and what is the fallback when they're unavailable for a month (judge drift goes uncorrected and live experiment verdicts degrade silently)?
- Model-snapshot lifecycle: how does the retirement-monitor surface an impending pinned-snapshot deprecation, and what is the explicit invalidation procedure for affected workflow_versions and (under a shared library) pages generated under the retired snapshot?
- Source-licensing posture: how do synthesis prompts + grounding rubric + a near-duplicate check enforce original re-expression with attribution (not verbatim reproduction) of scraped copyrighted material, and how are attributions surfaced as both a legal obligation and a learner trust signal?

## Key risks

- Quality/credibility collapse from fabricated concepts and flawed/inaccessible visuals — the existential risk; evals + learner 'report a problem' trust signals are the only defenses, and AI cannot self-supply the expert verification needed to match hand-built quality.
- Cost & pricing non-viability: ~$26 variable cost per 40-page curriculum (Critic loop ~45%, eval passes ~17%) makes per-curriculum consumer pricing a non-starter against $4/mo and free anchors — one curriculum eats six months of a $4 subscription. The segment/unit decision is existential.
- Statistical-power infeasibility on the long tail: most topics will never accumulate enough learners per variant for valid online learning-outcome conclusions (very roughly ~1,500/arm for a 5pp lift in aggregate, far more than any niche topic sees), forcing an offline-evals-first program with topic-bucketing and within-session surrogates rather than the full CUPED+sequential machinery as if traffic exists.
- Unit-of-generation lock-in: per-curriculum vs. shared-page-library drives cost, the Postgres schema, the A/B randomization unit, AND whether A/B is even powered — getting the schema un-share-ready in v1 is an expensive retrofit.
- Accessibility failure of generated artifacts: Canvas is invisible to assistive tech and SVG often is too; a WCAG-failing generated page is a hard procurement gate and legal exposure (Section 508 / EN 301 549 / ADA) for institutional buyers — chrome-level prefers-reduced-motion does NOT cover this.
- Content-moderation gap: a from-any-topic public generator can produce harmful/dangerous/age-inappropriate or authoritative-sounding-misinformation curricula that the quality evals (targeting fabrication and pedagogy) will happily pass; a separate, evaluable topic-admissibility gate is required.
- SME single point of failure: the eval+A/B program depends on a recurring pedagogy expert for the gold set, monthly judge recalibration, quarterly gold-set refresh, and an annotation queue — for a solo founder, an unavailable SME silently degrades every live experiment's verdict; budget it as a line item, not goodwill.
- Model/vendor drift: pinned dated model snapshots get retired (a near-certain event), silently invalidating in-flight experiments and frozen workflow versions and breaking the prompt cache (which roughly doubles cost); plus Vercel Workflow DevKit lock-in and unverified maturity behind the resumable-stream feature, which is a nice-to-have, not a v1 core dependency.
- Garbage-in curriculum graph: a wrong prerequisite DAG cascades difficulty cliffs into every page and is invisible to page-local evals; mitigated only by the pre-synthesis topo-sort/citation/coverage hard gate, and its LLM-feasibility is medium-confidence, not validated by Anthropic's research-system evidence.
- IP/source-licensing exposure behind the 'cited grounding' promise: reproduction of copyrighted scraped text/figures and unmet attribution/licensing obligations create legal and brand risk; treat scraped content as evidence to re-express and attribute, never as content to reproduce.
- Goodhart/proxy-optimization: an engagement-driven optimizer selects for fluent, low-difficulty, judge-pleasing material that harms durable learning (interleaving: 63% delayed vs 20% blocked), and delayed-retention instrumentation — the intended counter — is itself only reachable for head topics and opted-in returning learners.
- Cold-start / domain-coverage cliff: the interactive advantage may hold only for STEM, and thin-coverage niche topics return sparse retrieval — without explicit refuse/degrade-to-text/flag-low-confidence routing the pipeline fabricates exactly where it is most dangerous.

## Recommended next steps

- Build the generator skeleton end-to-end (durable-workflow spine + share-ready Postgres schema + SSE + sandboxed cross-origin iframe renderer) and ship ONE real 40-page curriculum to look at — you cannot write a meaningful rubric without concrete artifacts.
- Validate the §1 cost model against reality: run that one curriculum with count_tokens() instrumentation and per-stage cost logging; confirm the ~$26 figure, the Critic ~45% / eval ~17% split, and that the shared graph/brief prefix actually caches (cache_read_input_tokens > 0). Feed the real number into the pricing/segment decision before any pricing commitment.
- With the first few real curricula in hand, have the SME label the initial ~50-100 (topic x settings) gold set and FREEZE the binary rubric (incl. the accessibility and source-legitimacy dimensions); calibrate a cross-family judge to kappa>~0.6. From here the gold set gates every generator change — this is the offline-first decision surface that makes the low-traffic A/B story viable.
- Stand up the two hard, deterministic gates that most separate 'looks built' from 'is built and usable': a headless render check (no errors, viz does cognitive work) AND an axe-core/Lighthouse-a11y pass; a failing a11y or render check holds the page at 'soon'. Add an explicit accessibility contract to the synthesis spec so it's a generation target, not a retrofit.
- Implement the topic-admissibility classifier (allow / refuse-with-explanation / allow-with-constraints) as a discrete, versioned, evaluable component with its own small labeled set, running on topic+settings before any expensive generation; log model-native stop_reason:'refusal'+category as a secondary signal, not a substitute.
- Wire the cold-start path: compute coverage_confidence per node in grounding-verification, route thin nodes to degrade-to-text or 'soon', and surface near-zero-coverage topics at the §3 review gate (refuse/narrow) instead of fabricating.
- Instrument the output side early and cheaply: the fixed-string event schema, IntersectionObserver/Page-Visibility/sendBeacon recipe baked into the page template, learner-facing trust signals (visible citations/attributions, coverage_confidence badge, 'report a problem' button feeding the annotation queue), and a returning-learner pseudo-ID + consent_status that makes delayed-outcome joins possible later.
- Build the model-snapshot-retirement monitor (scheduled Models-API check of every snapshot referenced by an active workflow_version, alert before retirement) plus the cache_read_input_tokens>0 assertion, and define the explicit invalidation procedure for affected experiments and shared pages — before any live experiment depends on a pinned snapshot.
- Defer the full A/B experimentation program until traffic exists; in the interim rely on offline evals + interleaving for ranking decisions, and design any future online learner-outcome experiment with a pre-registered horizon, a manufactured pre-period covariate (the §3 diagnostic) for CUPED, topic-bucketing for the long tail, and the honest expectation that many reads will be inconclusive.
- Capture the source-licensing guardrails now: record each source's URL + detectable license/robots posture, add the 'original re-expression, no verbatim block-copy' instruction to synthesis prompts, and add a deterministic near-duplicate check against retrieved text as a hard guard.
